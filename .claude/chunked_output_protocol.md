# 分块输出协议

> 《龙珠：厄日序曲》创作协作专用

---

## 协议目的

避免API输出长度限制导致章节内容截断丢失。

---

## 触发条件

当生成以下内容时，必须启用分块输出：
1. 完整章节草稿（通常2000-4000字）
2. 大段修改内容（超过1500字）
3. 多场景连续输出

---

## 分块规则

### 1. 章节输出分块

**标准章节（2000-4000字）分为2-3块：**

```
【第X章 - 第1部分】
[场景1-2的内容]
---继续---

【第X章 - 第2部分】
[场景3-4的内容]
---继续---

【第X章 - 第3部分】
[场景5+结尾的内容]
---完成---
```

### 2. 分块边界选择

优先在以下位置分块：
- 场景切换处（【地点：XXX】标记前）
- POV切换处
- 时间跳跃处
- 自然段落结束处

**禁止**在以下位置分块：
- 对话中间
- 动作描写中间
- 情绪高潮处

### 3. 分块标记格式

```
---继续---     # 表示还有后续内容
---完成---     # 表示章节结束
```

---

## 执行流程

1. 评估内容总长度
2. 规划分块点（按场景/POV划分）
3. 输出第一块 + `---继续---`
4. 等待确认后输出下一块
5. 最后一块使用 `---完成---`

---

## 用户指令

- `继续` - 输出下一块
- `重写本块` - 重新生成当前块
- `跳过` - 跳过当前块，继续下一块
